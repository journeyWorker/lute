# lute-check 0.1.0 Cutover Implementation Plan (Plan B of 6)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Bring the checker up to the full DSL 0.1.0 semantics: real `<hub>` checking (replacing the transitional `E-HUB-UNSUPPORTED` gate), `<when is>` literal-pattern coverage, and the remaining 0.1.0 static diagnostics — leaving the workspace green with hub/`is`/interpolation documents that pass `lute check`.

**Architecture:** All work in `crates/lute-check` (+ small parser/compile touches only for the `as`→`into` atomic rename, Task B8). The checker is one `check()` pipeline (`check.rs`) that folds a `StateSchema`, walks nodes (`Walker`), and delegates to `match_check.rs` (match/branch exhaustiveness + recording), `defassign.rs` (path-sensitive definite assignment), `cel_resolve.rs` (`@ref`/`$`/state-path checks), `timeline.rs`, `set_op.rs`, `meta.rs`. Plan A left `Arm::When.is: Option<IsPattern>` in the AST (Task 11) and `<hub>` parsed with `once`/`exit` as bare choice attrs.

**Tech Stack:** Rust; spec = `docs/proposals/scenario-dsl/0.1.0.md` ("dsl §N"); design contract `docs/superpowers/specs/2026-07-06-lute-dsl-0.1.0-design.md`; diagnostic registry = spec Appendix E.

## Global Constraints

- Diagnostic spellings verbatim (spec Appendix E): `E-HUB-NO-EXIT`, `E-WHEN-PATTERN`, `E-PATH-IDENT`, `E-BRANCH-ALL-GUARDED`, `E-AT-CONTEXT`, `E-CLIP-TIMING`, `E-TIMELINE-DURATION`, `E-UNSET-UNCOVERED`, `E-CHOICELOG-READ`, `E-PERSIST-MISSING-INTO`. Remove `E-HUB-UNSUPPORTED` (transitional) when B6 lands.
- Severity is binary (`Error`/`Warning`); no Info tier.
- `check()` is pure/total; every diagnostic carries an original-source span; diagnostics are collected, deduped, and sorted `(byte_start, code)`.
- Run only the crate(s) a task touches; `cargo test --workspace` gates the final task. Do NOT run project-wide fmt/clippy per task.
- Reuse existing machinery: `analyze_arm`/`infer_domain`/`is_exhaustive` (match_check), `apply_condition`/`check_reads` (defassign), `check_cel_slot`/`scan_refs` (cel_resolve). Extend, don't reinvent.
