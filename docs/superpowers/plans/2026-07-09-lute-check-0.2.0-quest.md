# lute-check 0.2.0 — Document Kinds & the Quest Kind (Plan C of 5)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `lute-check` validate the 0.2.0 document-kind system + quest kind: the `kind:` discriminator, per-kind grammar admission (`E-GRAMMAR-NOT-ADMITTED`), the `quest.*` state tier, `<on>` ECA semantics, quest/objective identity + folded reserved state, and the reserved-write policy — replacing Plan A's transitional `E-QUEST-UNSUPPORTED` gate with real semantics.

**Architecture:** The checker is the ONLY layer that knows about `kind:` (the parser is kind-agnostic; Plan A). `check()` resolves a `DocKind {Scene, Quest}` from frontmatter and dispatches: scene = the existing shot walk (unchanged); quest = a parallel walk over `doc.quests`. Grammar admission is a NEW **table-driven** model (a `NodeKind`/`GrammarContext` classifier + one exhaustive `admits(...)` fn) — the crate's first, deliberately chosen so 0.3.0's `::assert`/`::retract` slot in as one arm and the compiler forces every dispatch site. The `quest.*` tier reuses the existing scalar `Namespace`/`STATE_ROOTS`/schema/defassign machinery via a single new `Namespace::Quest` + `STATE_ROOTS += "quest"`. `check_quest`/`check_objective` mirror `check_branch`/`check_hub` to fold reserved implicit decls. `<on>` reuses the `<hub>` may-write defassign join. The `::set` write-policy becomes a small reusable `WriteOwner` classifier (0.3.0 extends it for relations).

**Tech Stack:** Rust (workspace `cargo test`), spec = `docs/proposals/scenario-dsl/0.2.0.md` (cited "dsl 0.2.0 §N"), design = `docs/superpowers/specs/2026-07-07-lute-dsl-0.2.0-design.md`. Depends on Plan A (AST) + Plan B (`events` export) being committed.

## Global Constraints

- Diagnostic codes VERBATIM (dsl 0.2.0 Appendix B): `E-KIND-MISSING`, `E-UNKNOWN-KIND`, `E-GRAMMAR-NOT-ADMITTED`, `E-ON-NO-EVENT`, `E-UNKNOWN-EVENT`, `E-QUEST-ID-DUP`, `E-OBJECTIVE-MISSING-DONE`, `E-OBJECTIVE-ID-DUP`, `E-QUEST-RESERVED-WRITE`. Reuse (unchanged): `E-META-MISSING`, `E-META-UNKNOWN-KEY`, `E-UNDECLARED`, `E-MAYBE-UNSET`, `E-CEL-PROFILE`, `E-PATH-IDENT`, `E-NONEXHAUSTIVE`/`E-UNSET-UNCOVERED`, `E-L10N-PLACEHOLDER`. Remove the transitional `E-QUEST-UNSUPPORTED` (Plan A).
- **0.3.0-forward (build seams, not the feature — YAGNI):** grammar admission MUST be a table-driven `admits(DocKind, GrammarContext, NodeKind) -> bool` (exhaustive `match`, no wildcard on `NodeKind`) so 0.3.0 adds `Assert`/`Retract` as one arm and the compiler flags every unhandled site. The `::set` write-policy MUST be a reusable `WriteOwner` enum + `classify_write(path,schema)->WriteOwner` (0.3.0 adds relation owners). `Namespace`/`STATE_ROOTS` stay the tier-GENERAL vocabulary (0.3.0 relations reuse the same tiers). on/objective/quest CEL guards MUST flow through the existing `check_cel_slot` profile gate (0.3.0's `now()`/`holds`/`count`/`validAt` become available for free). Keep `parse_meta_kind` section handling modular. Prefer exhaustive enums + compiler-forced completeness over stringly/wildcard checks.
- Built-in lifecycle events come from `lute_manifest::snapshot::BUILTIN_LIFECYCLE_EVENTS` (Plan B) — do NOT redefine the list here. `E-UNKNOWN-EVENT` passes when `event ∈ BUILTIN_LIFECYCLE_EVENTS ∪ snapshot.events.keys()`.
- The `quest.<id>.state` enum domain is `[active, complete, failed]` (∪ implicit `unset` coverage); `quest.<id>.objectives.<oid>.done: bool` default `false`. These are reserved implicit decls (`Namespace::Quest`), engine-populated, author-unwritable.
- Kind is resolved BEFORE per-kind meta keys. Import-role docs (Schema via `uses:`, Component via `components:`) MUST NOT carry `kind:` (dsl 0.2.0 §3.1) and are NEVER Scene/Quest.

### Task 5: Table-driven grammar admission — `E-GRAMMAR-NOT-ADMITTED`

**Files:**
- Create: `crates/lute-check/src/admission.rs` (new module)

**Interfaces:**
- Produces (the 0.3.0-forward table-driven model):
  - `pub enum NodeKind { Line, Directive, Set, Branch, Match, Hub, Timeline, On, Objective }` + `fn node_kind(&Node) -> NodeKind` (exhaustive — the compiler-forced canary for future variants).
  - `enum GrammarContext { SceneBody, QuestBody, Emittable }` — `SceneBody` = anywhere inside a scene shot; `QuestBody` = directly inside a `<quest>`; `Emittable` = inside an `<objective>`/`<on>` body OR inside any `<match>`/`<branch>` nested within a quest doc.
  - `fn admits(doc: DocKind, ctx: GrammarContext, nk: NodeKind) -> bool` — an EXHAUSTIVE `match` over `(ctx, nk)` (no `_` on `nk`). SceneBody admits Line/Directive/Set/Branch/Match/Hub/Timeline (dsl 0.1.0 set); QuestBody admits Line/Directive/Set/Branch/Match/On/Objective; Emittable admits Line/Directive/Set/Branch/Match. Hub/Timeline never admitted in a quest doc; On/Objective never admitted in a scene doc.
  - `pub fn check_admission(doc: &Document, kind: DocKind) -> Vec<Diagnostic>` — walks the document, emitting `E-GRAMMAR-NOT-ADMITTED` (at the offending node's span, with a message naming the construct + why) for: (a) scene doc with a non-empty `doc.quests` (a `<quest>` where scene forbids it); (b) quest doc with a non-empty `doc.shots` (a `## ` heading / shot — quest forbids headings); (c) any `Node` whose `NodeKind` is not admitted by its `(kind, context)`. Context transitions: scene body stays SceneBody through branch/match/hub/timeline recursion; quest `<quest>` body is QuestBody, and descending into an objective body / on arm / any match arm / branch choice switches to Emittable (so a nested `<objective>`/`<on>` inside those → not admitted).
- Consumes: `DocKind` (Task 1), the AST.
