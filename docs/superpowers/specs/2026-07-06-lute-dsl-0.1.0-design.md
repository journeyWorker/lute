## D-A. Content line (hot-path cost) — BREAKING

```
Line ::= ":" Speaker Attrs? ":" WS Text
```

- `:bianca{code="0010" emotion="delighted"}: Mr. Fixer!` / `:narrator: Venny's again.`
- `:line[...]` form is REMOVED (clean cutover). Line classification precedence becomes:
  1. `## ` shot heading, 2. `# ` title, 3. `::set{`, 4. `::` ident, 5. `:` ident,
  6. `<` / `</`, 7. error.
- Migration note in spec: implementations SHOULD detect a leading `:line[` and emit a
  fix-it to the 0.1.0 form.
- `<choice>` `as` attr is RENAMED `into` (persist sugar: `persist="run"
  into="run.metHelpfully" value="warm"`); `as` remains only on content lines
  (display-label override). Diagnostic rename: `E-PERSIST-MISSING-AS` →
  `E-PERSIST-MISSING-INTO`.

## D-B. Interpolation `{{…}}` — NEW

- Grammar (inside `Text` and inside `<choice label>` string values):
  `Interp ::= "{{" WS? (Path | Ref | ReservedToken) WS? "}}"`,
  `ReservedToken ::= "userName"`.
- `Ref` = `@def` / `@fn(args)` — defs ARE allowed (user decision): undeclared refs are
  caught by existing `E-UNDECLARED-REF` machinery; undeclared paths by `E-UNDECLARED`;
  an interpolation read is a READ for definite-assignment (`E-MAYBE-UNSET` applies).
- Rendering (normative): number → shortest decimal; bool → `true`/`false`; enum →
  member text verbatim.
- Escapes: literal `{{` is written `\{{`; an unterminated `{{` is `E-INTERP-UNTERMINATED`.
- i18n contract: placeholders are part of the line's translatable text keyed by
  `lineId`; at build time the compiler MUST check source↔translation **placeholder-set
  equality** (`E-L10N-PLACEHOLDER`, compile-stage). ICU plural/select is an explicitly
  reserved section (non-goal for 0.1).

## D-C. `<hub>` — NEW (Ink-parity revisit conversations)

```
Hub       ::= "<hub" Attrs ">" HubChoice+ "</hub>"        (* requires id *)
HubChoice ::= "<choice" Attrs ">" Node* "</choice>"
              (* requires id, label; flags once, exit; optional when, persist/into/value *)
```

- Semantics: on entry and after each non-`exit` arm completes, re-present all eligible
  choices. Eligible = `when` (if any) true ∧ (if `once`, not yet taken). Taking an
  `exit` choice runs its arm, then leaves the hub. If no choice is eligible at a
  presentation point, the hub ends (auto-exit). An `exit`-flagged choice may also be
  `once` (legal; `once` is simply irrelevant after exit).
- Static guarantee `E-HUB-NO-EXIT`: a hub MUST have at least one **unguarded** (no
  `when`) `exit` choice, OR all of its choices MUST be `once`.
- Recording (implicit, episode-scoped):
  - `scene.choices.<hubId>` = last selected choice id (enum: choice ids ∪ `unset`);
  - `scene.visited.<hubId>.<choiceId>: bool` (default `false`) — NEW reserved implicit
    namespace `scene.visited.*` (separate from `scene.choices.*` to avoid path
    parent/leaf collision).
- `id` uniqueness: hub ids and branch ids share one per-episode uniqueness domain
  (both record under `scene.choices.*`); choice ids unique within a hub
  (`E-CHOICE-DUP` reused).
- Definite assignment: hub arms have NO dominance relation (same join rule as match
  arms); arm writes are may-writes at hub exit.
- Totality argument (goes into DSL §3): a hub reduces at build time to finitely many
  records; runtime re-presentation consumes player input each cycle, so no unbounded
  *computation* is introduced. The §3.2 "reduces to data" invariant is preserved.
- Weave coverage: nesting (`<branch>`/`<match>` inside choice arms) already exists;
  hub adds revisit. Backward diverts remain a non-goal (see §NG).

## D-D. `<when is>` literal patterns — NEW

- `When` requires `is` and/or `test` (`E-WHEN-PATTERN` when neither).
- `is` value: one literal or `|`-alternation — enum member idents, `true`/`false`,
  decimal numbers, and `unset`. Matching is equality on the match subject; `unset`
  matches the unset state.
- `is` + `test` together = pattern AND guard (ts-pattern `P.when` composition).
- Exhaustiveness (normative): coverage is computed from `is` arms (plus `otherwise`).
  Recognizing provable CEL `test` equalities is downgraded to MAY. Omitting
  `<otherwise>` remains the exhaustive assertion (≡ ts-pattern `.exhaustive()`); a
  correspondence note goes in the spec.

## D-E. Lute-CEL profile — NEW normative section (fixes S1, S2, S14)

- **Numeric model:** state `type: number` is IEEE-754 double (the compile target is
  JSON). In Lute CEL slots every numeric literal denotes a double (integer literals
  promote). `%` is not available in 0.1 (no integer domain); arithmetic is `+ - * /`.
- **Environment (closed):** CEL operators/literals/lists/ternary, the `in` operator,
  the `has()` macro, and the Lute extension `isSet(<path>)`. No other functions in 0.1.
- **Hygiene (normative):** `@ref` expansion and `$` substitution are parenthesized,
  AST-safe substitutions: an expanded def body and every bound argument MUST be
  wrapped in `(…)`; a compound `$` subject MUST be substituted parenthesized (bare
  dotted paths may go verbatim). This codifies shipped `lute-compile/src/expand.rs`.
- **Identifier alignment (S2):** `-` is FORBIDDEN in CEL-facing names — state-path
  segments, def names, def params (`E-PATH-IDENT`). Directive names, attribute keys,
  speaker ids, choice/branch/hub ids, and asset ids keep permitting `-`.

## Plugin-system errata (stays 0.0.1, Errata appendix)

1. `plugins: {id: <non-map, non-true>}` (incl. `false`) is a **static error**
   (`E-PROFILE-PLUGIN-VALUE`) — never a silent activation. Activation stays additive;
   explicit disable remains future work.
2. Register reality: `enums` export kind added to §4/§6 closed set; `{assetKind:
   <name>}` added to §7 `Type`.
3. `depends[].range` grammar pinned: caret ranges (`^x.y.z`, pre-1.0 caret semantics)
   and exact versions only; anything else is unsatisfiable by definition.
4. NEW conformance appendix listing the four spec'd-but-unimplemented MUSTs as
   conformance items: §5.1 option-name/type validation, §6.8 frontmatter value schema
   validation, §9 stale-default (result-read-before-produced) diagnostic, §14 reserved
   timing-attr redefinition rejection. Also: cross-plugin duplicate diagnostics MUST
   name the first owner (no `"?"` placeholders).

## Character-cast errata (stays 0.0.1, Errata appendix)

1. Remove the undefined term "authored alias" (§3.1, §7.3) — player normalization is
   defined solely in terms of the `pov` speaker id.
2. Lowering precedence rule: a plugin directive whose manifest declares `lower: {kind:
   set}` desugars to `set` records at compile time; this takes precedence over the
   generic non-core→`plugin` passthrough. `::seal`/`::reveal`/`::wear` declare it.
3. Non-player `delivery="thought"` legal (aligns with X1); monologue is no longer
   defined as player-only.
4. voiceBank static determinability: costume resolution reuses the compiler's
   conservative CFG join (compile-IR D9); a path-joined costume that is factually
   identical MAY still be conservatively rejected — documented as expected behavior.
5. Costume rendering contract: current costume applies to dialogue sprites; the IR
   carries it (see IR addendum) — the engine does not reconstruct it from `set` records.
6. Syntax examples updated to 0.1.0 content-line form.

## Compile-IR addendum (new section in the 2026-07-04 design doc)

1. `sprite` records gain `costume`; stage fork/join slots become anchor/pose/emotion/
   **costume**.
2. NEW `hub` record kind: `{id, options: [{id, label(lineId), once, exit, when?,
   target}], converge}`; runtime eligibility reads `scene.visited.<hubId>.*`; selection
   writes `scene.choices.<hubId>` and the visited flag.
3. `line` records: `text` remains the source text; NEW `placeholders: [ … ]` listing
   each interpolation (path/ref/reserved token, in order) for engine substitution and
   translation-fill validation.
4. Envelope `meta.episodeId` becomes the lineId derivation input (S3); choice-label
   lineId derivation confirmed episode-prefixed (this doc was already correct).
5. role derivation per X1 (reference the DSL table; delete any divergent local wording).

## NG. Non-goals (spelled out in scenario-dsl 0.1.0 §Non-goals)

- **Opaque lineId mode** — deferred, not rejected: with `episodeId` pinning the
  remaining breakage (recast, cross-episode line moves) is either semantically a new
  unit or rare; opaque ids would cost pipeline readability (VO file naming, string
  tables). Reserved as a future `lute tag` derivation option (`lineIdMode: opaque`) —
  a tooling change, not a language change.
- **Backward diverts** — permanent non-goal: cycles would demote every static analysis
  (definite assignment, exhaustiveness, stage join, path enumeration) to fixpoint
  computations and make termination author-owned. `<hub>` covers the legitimate
  revisit patterns structurally. Reserved constraint: any future divert MUST be
  forward-only (DAG-preserving), e.g. a `::goto{shot=N}` with N strictly ahead.
- **ICU MessageFormat** — reserved section only (placeholder-set contract ships now).
- Kept as-is (complexity judged already minimal): 4-tier state model, timeline
  `<track>` syntax, YAML frontmatter, `::set` statement form, no `/=` operator.
