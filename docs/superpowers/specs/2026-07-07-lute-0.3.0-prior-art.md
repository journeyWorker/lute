# Lute 0.3.0 — Prior-art research note (SVO fact-DB + Datalog)

- **Date:** 2026-07-07
- **Status:** research note (informs `docs/proposals/scenario-dsl/0.3.0.md`)
- **Question:** what can the 0.3.0 relational layer — a **closed SVO fact database + total
  Datalog-style derivation** — learn from four internal sibling systems?
- **Systems studied (read-only):** `bard`'s **nekg** (Narrative Entity Knowledge Graph),
  **bard-chronicle** (character memory query engine), the **lens/POV** system, and
  `rnd-flutter`'s **feature/spec/ledger** parity harness.

## TL;DR

`bard` already built ~80% of what 0.3.0 theorizes, and its failure modes are *exactly* what
Lute's total/closed/statically-verifiable design prevents:

- **nekg = the fact atom + the cautionary control group.** A pure SPO triple with a
  bitemporal validity interval — but with **open free-text predicates/entities**, **manual
  (LLM) accumulation**, and **convention-only invariants**. All three rotted in production
  (a `fix-nekg-facts.ts` canonicalization script, dropped facts from EP04, per-writer
  duplication). Lute's declared-closed-schema + engine-derived accumulation + compile-time
  checks are the direct fixes.
- **bard-chronicle = the blueprint, rebuilt total.** Facts as an **append-only log with
  tombstone intervals** + a **pure-recompute derived view** + a **POV projection**. The
  shape is right; its derivation is dedup+filter (no real joins/recursion) and its
  invariants are offline-lint-only. Lute supplies the missing piece: **bounded terminating
  Datalog with compile-time guarantees.**
- **lens/POV = epistemic modeling.** "Who knows what" as **observer-as-subject SVO** +
  **projection over one shared fact-set**. Adopt the shape; reject its imperative
  heuristics (substring tables, float thresholds) and its LLM-terminal and
  anti-convergence design goals.
- **rnd-flutter ledger = governance.** Provenance pins, append-only per-key event log,
  **absence-of-record = the violation signal**, closed change-controlled diagnostic
  vocabulary, one adversarial fixture per invariant, policy-as-data, and **never rank by an
  author-supplied field**.

**Meta-lesson (vindicates Lute's whole thesis):** every convention-only invariant in these
systems rotted. 0.3.0's schema closure, rule termination, arity, and derivation bounds MUST
be **compile-time structural guarantees**, never an optional post-hoc lint.

## How this maps onto Lute's total/Datalog frame

- **Closure is doubly free in a game.** Datalog is total because it is function-free (finite
  Herbrand base). A game adds a second closure: facts are drawn from a *declared finite
  schema*, not free text (nekg's open vocab is precisely the missing second closure).
- **Append-only + pure recompute = trivially correct replay/save-scum.** chronicle proves
  this; Lute gets it by defining derivation as a pure function of the asserted tuples up to
  the current tick.
- **The differentiator has no prior art.** None of the four systems has real
  joins/transitive-closure/rule-evaluation — "derivation" everywhere is dedup+filter or a
  graph heuristic. Lute's bounded, terminating, statically-verifiable Datalog is the net-new
  contribution; borrow only the *stage separation* (temporal validity → gating → projection)
  as a consumption pattern, not the engines themselves.

## Deferred / non-goals (confirmed by the survey)

- **Prolog power** (function symbols, unbounded recursion) — the only genuinely non-total
  part; rejected. Datalog is the ceiling.
- **LLM-in-the-loop** resolution of what is "known"/true — violates engine-owns-runtime.
- **Approximate/heuristic scoring** (PageRank, float thresholds) as a query primitive.
- **Anti-convergence epistemics** (perpetually-non-convergent belief vectors) — needs
  unbounded/unverifiable state.
