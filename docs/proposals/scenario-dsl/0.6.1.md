# Lute Scenario DSL — 0.6.1: Verification-Coverage Signals & Warning Promotion

- **Version:** 0.6.1
- **Status:** Draft (**compatible refinement** of 0.6.0)
- **Date:** 2026-07-15
- **Scope:** three warning-grade coverage signals and one CLI facility, all
  motivated by AI-pipeline review (2026-07-15): pipelines judge by exit code,
  so verification GAPS must be visible signals, and a harness must be able to
  promote any warning to an error. No grammar productions, no IR shape change,
  no `decide()` (§5.1 R1–R5) change, no Closure change.

## 1. Motivation

`check` is sound but deliberately incomplete over the relational layer: a
fact query (`holds`/`count`/`validAt`) over a **producible** relation is
always `Undecided` (R5), so a fact-gated objective passes `check` clean
whether or not it is ever satisfiable in play. `producible()` (0.4.0 §4.2,
connectivity T7) already closes the never-producible half with
`E-OBJECTIVE-UNSATISFIABLE`; the producible half is **unprovable by design**
— and that boundary lives only in `architecture.md` prose. A human reads the
docs; an AI pipeline reads exit 0. These signals put the boundary on the tool
surface: which regions are proof-covered, which are review-covered.

## 2. `W-UNPROVEN-RELATIONAL` (new warning)

Fires once per `<objective done>` / `<quest start|fail>` predicate whose CEL
contains at least one fact-query call over a relation `producible()` judges
producible (the never-producible case stays `E-OBJECTIVE-UNSATISFIABLE`,
0.4.0 §4.2, and is NOT double-reported). Message names the queried
relation(s) and the boundary:

> `W-UNPROVEN-RELATIONAL` — `done="holds(inParty(shadowheart))"` is gated by
> a relational fact query; static reachability analysis (§5) neither proves
> nor refutes it. Verify with `lute trace` seeds or human review.

Severity **warning**, `Layer::Logic`. This is a COVERAGE signal, not a
defect report: it marks the objective as outside the proof envelope. A
pipeline that must not auto-pass review regions promotes it (`--deny
W-UNPROVEN-RELATIONAL`, §5). Scalar-only gates never fire it.

## 3. `W-LUTE-VERSION-STALE` (new warning)

Fires when frontmatter `luteVersion` is present and differs from the
toolchain's `LUTE_LANG_VERSION`. D13 stands: `luteVersion` remains a
universal key, never validated against capabilities — this is a
warning-grade freshness signal only (catches a model reproducing a stale
stamp from a copied example). Message names both versions and the fix
(`luteVersion: "<current>"`).

## 4. `W-TRACE-MOCK-UNPRODUCIBLE` (new trace warning)

`lute trace` mock validation (`E-TRACE-MOCK-FACT`'s pass, 0.4.0 §4.3)
additionally warns for a `--fact`/mock-YAML fact whose relation
`producible()` judges NOT producible: the supplied answer can never arise
from authored producers, so a "complete" walk seeded with it proves nothing
about reachable play. Warning only — a mock is a hypothesis and D1 stands
(the CHECKER's `producible()` result is computed at build-input time and
carried into trace; trace itself still runs no Datalog). A `reserved: true`
or `open: engine`-argument relation is producible by definition (0.4.0 §4.2)
and never warns.

## 5. CLI: `--deny <CODE>` / `--deny-warnings` (tooling)

`lute check` and `lute check-project` gain:

- `--deny <CODE>` (repeatable): every diagnostic with exactly that code is
  treated as an error for the verdict and exit code (severity reported as
  error in both human and `--json` output, with a `denied: true` marker in
  JSON so the promotion is distinguishable from a native error).
- `--deny-warnings`: promotes every warning.

rustc/clippy precedent (`-D`). An unknown code in `--deny` is a usage error
(exit 2) — a typo'd promotion MUST NOT silently protect nothing. Compile and
trace are unchanged (their gates are already error-driven).

## 6. Non-goals

- No `decide()` or Closure change; R5's firewall stands.
- No fact producer/consumer graph view (`lute scenario` extension) — a
  separate proposal if wanted.
- No `--allow` demotion; errors are never demotable.

New warnings: `W-UNPROVEN-RELATIONAL` (§2), `W-LUTE-VERSION-STALE` (§3),
`W-TRACE-MOCK-UNPRODUCIBLE` (§4). No codes removed; no clean document's
verdict changes except under explicit `--deny` promotion.
