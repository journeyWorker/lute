# Lute Character & Cast ‚Äî Capability Proposal

- **Version:** 0.0.1
- **Status:** Draft (pre-implementation).
- **Scope:** the normative **language contract** for character identity, display label, costume,
  name-reveal, and voice-join in a `.lute` scenario ‚Äî as a **capability plugin** (`character/cast`).
  Covers the character **registry** (provider), the `cast:` **frontmatter**, the `scene.cast.<id>.*`
  **state**, the compile-time **resolution** of sprite / display-name / voiceKey, the `seal`/
  `reveal`/`wear` **directives**, and **migration** of legacy content.
- **Out of scope:** the compiler/runtime **implementation** (deferred; a future Rust core ‚Äî this
  document specifies *what* it must produce, not *how*). The design **rationale + audit trail** is
  [`design.md`](design.md).
- **Depends on:** Lute Scenario DSL `^0.0.1` ([`../scenario-dsl/0.0.1.md`](../scenario-dsl/0.0.1.md))
  with the ¬ß6.1 *plugin-contributed meta keys* amendment and the `as` line attribute; the Lute
  Plugin System `^0.0.1` ([`../plugin-system/0.0.1.md`](../plugin-system/0.0.1.md)) with the
  `frontmatter` export.
- **Reference example:** [`../../examples/date-minigame.lute`](../../examples/date-minigame.lute)
  (cast usage to be reflected there when examples are refreshed).

The key words **MUST**, **MUST NOT**, **REQUIRED**, **SHOULD**, **SHOULD NOT**, **MAY** are to be
interpreted as in RFC 2119.

---

## 1. Abstract

A `:line[<id>]` bracket carries **only** the stable `characterId` (identity ‚Äî the key for sprite
lookup, voice/line joins, and state). Three further concerns are modelled separately, never packed
into the bracket:

1. **display label** ‚Äî the name the dialog box shows (`???`, a mask, or the real name);
2. **costume** ‚Äî an appearance variant of the same identity (`CH.{id}.{costume}.{emotion}.{variant}`);
3. **name-reveal** ‚Äî the `???`‚Üíreal-name transition, modelled as first-class state that flips once
   and affects all later lines.

All mutable cast state is `enum`/`bool` (satisfying the language's `number|bool|enum` state types,
`../scenario-dsl/0.0.1.md` ¬ß9.3); label strings are **static** configuration; and everything reduces
to flat command records + CEL, preserving the language invariants (total, non-Turing).

## 2. Relationship & dependencies (normative)

This capability is a plugin. It REQUIRES:

- **Language ¬ß6.1 plugin-contributed meta keys** ‚Äî so `cast:` is a legal frontmatter key owned by
  this plugin.
- **Language `:line` `as` attribute** ‚Äî the one-off display-label override (¬ß6).
- **Plugin-system `frontmatter` export** ‚Äî so this plugin can declare and validate the `cast:` key
  (`../plugin-system/0.0.1.md`).
- **Plugin-system providers, state templates, directives, and named lowering hooks** ‚Äî for the
  registry (¬ß3), the `scene.cast.<id>.*` template (¬ß5), and the directives/resolution (¬ß7‚Äì¬ß8).

A conforming consumer (checker/LSP/compiler) MUST validate a document against the single resolved
capability snapshot that includes this plugin.

## 3. Character registry (provider)

The registry is global, static character identity data, supplied as a plugin **provider**
(snapshot-first, `../plugin-system/0.0.1.md` ¬ß10). Per character id:

```yaml
bianca:
  name: { ko: "ÎπÑÏïôÏπ¥", en: "Bianca" }   # REQUIRED ‚Äî canonical display name per language
  costumes: [default, waitress, casual]    # REQUIRED ‚Äî costume domain; MUST include `default`
  defaultCostume: default                  # REQUIRED ‚Äî MUST be a member of `costumes`
ann:
  name: { ko: "Ïï§", en: "Ann" }
  costumes: [default, child]
  defaultCostume: default
  costumeVoiceBank: { child: ann_child }   # OPTIONAL ‚Äî per-costume static voice bank (¬ß7.3)
```

| Field | Req. | Rule |
|---|---|---|
| `name` | MUST | map from language code ‚Üí display string; MUST cover the project's languages |
| `costumes` | MUST | non-empty list of idents; MUST include `default` |
| `defaultCostume` | MUST | ‚àà `costumes` |
| `costumeVoiceBank` | MAY | map costume ‚Üí a static voice-bank id (migration / voice-distinct costumes, ¬ß7.3) |

The `character` id domain is owned by this provider. A `:line`/`::auto`/`cast:` reference to an id
absent from a **fresh** registry snapshot is a static error; absent from a **stale** snapshot is a
*catalog-stale* diagnostic, not a hard error (`../plugin-system/0.0.1.md` ¬ß10).

## 4. `cast:` frontmatter (plugin-owned)

`cast:` is a frontmatter meta key owned by this plugin (legal via language ¬ß6.1 + plugin-system
`frontmatter` export). It seeds per-episode initial per-character state and static labels:

```yaml
cast:
  bianca:
    costume: waitress                 # OPTIONAL enum ‚àà registry.costumes; default registry.defaultCostume
    sealed: true                      # OPTIONAL bool; default false
    maskAs: "ÏùÄÎ∞úÏùò Ï¢ÖÏóÖÏõê"           # OPTIONAL static string; default "???"
  takeru: { costume: default }         # sealed omitted ‚Üí false (real name); maskAs unused
```

- Each key MUST be a `character` id present in the registry (¬ß3); an unknown id is a static error.
- `costume` (if present) MUST be a member of that character's `registry.costumes`.
- `sealed` and `costume` seed the corresponding **state** (¬ß5). `maskAs` is **static** (not state).
- A character not listed in `cast:` uses defaults (`costume = defaultCostume`, `sealed = false`,
  `maskAs = "???"`).

## 5. Cast state (normative)

This plugin declares, as a state template keyed by the registry's `character` id domain
(`../plugin-system/0.0.1.md` ¬ß6.3/¬ß7.4), for every referenced character `<id>`:

- `scene.cast.<id>.costume : enum[<registry.costumes for id>]` ‚Äî default `registry.defaultCostume`,
  overridden by `cast:` (¬ß4).
- `scene.cast.<id>.sealed : bool` ‚Äî default `false`, overridden by `cast:` (¬ß4).

Both are `enum`/`bool`, satisfying the language state types (¬ß9.3). They are **`scene`-scoped**
(episode lifetime; reset at episode end, re-materializing defaults). Because `sealed` defaults
`false`, a later episode that omits a character from `cast:` shows the real name automatically;
cross-episode "already known" needs no persisted flag. A rare multi-episode masked arc MAY bind the
mask to a declared `run.*` fact instead (author-declared; outside the auto-declared template).

`maskAs` is **static** per-character-per-episode data from `cast:` (or the default `"???"`); it is
**not** a state path and MUST NOT be written by `::set`.

Cast state is mutated only by `::set` (`../scenario-dsl/0.0.1.md` ¬ß7.3.4) or its sugar (¬ß8). No
`unset`/`null`/delete operation is introduced.

## 6. `:line` display-label override (`as`)

The language `:line` gains an OPTIONAL `as` attribute (a `String`): a one-off display-label override
for that single line. It does not change state. It has the highest precedence in name resolution
(¬ß7.2). Absent `as`, the resolution falls to state + registry.

```lute

```

## 7. Resolution (normative)

For each `:line[<id>]{code, emotion?, variant?, as?}` (and, for sprite/costume, each `::auto`), a
conforming compiler MUST resolve three outputs.

### 7.1 Sprite assetId
`CH.{id}.{costume}.{emotion}.{variant}`, where `costume` is the character's **current**
`scene.cast.<id>.costume` at that point (carried in stage state), `emotion` defaults to the prior
sprite emotion / `neutral`, and `variant` defaults to `0`. The current costume MUST apply to
**dialogue** sprites as well as `::auto`, not only at the point costume is set.

### 7.2 Display label
Precedence, first match wins:

1. the line's `as` attribute (¬ß6), if present;
2. else if `scene.cast.<id>.sealed == true` ‚Üí `maskAs` (the static label; default `"???"`);
3. else ‚Üí `registry.name[lang]` (the canonical name for the active language).

### 7.3 voiceKey (path-stable)
`voiceKey = {voiceBank}-{code}` where `voiceBank` is a **static, compile-time constant**:

- `registry.costumeVoiceBank[costume]` **only when** the character's costume is statically
  determinable at that line (from `cast:` or a dominating static assignment);
- otherwise `characterId`.

`voiceKey` MUST NOT be derived from mutable `scene.cast.<id>.costume` in a way that varies by
branch/timeline path ‚Äî voice joins MUST be path-stable. (Sprites MAY vary per path; they are not
join keys.) If a coded line's `voiceBank` would be path-dependent or ambiguous, it is a static error
(¬ß9); the author pins it or accepts `characterId`. With no `costumeVoiceBank`, `voiceKey =
{characterId}-{code}`.

## 8. Directives (plugin-provided sugar)

Thin, pure desugaring to `::set` (¬ß7.3.4) ‚Äî no new semantics; the primary authoring surface:

| Directive | Desugars to |
|---|---|
| `::seal{character="bianca"}` | `::set{scene.cast.bianca.sealed = true}` |
| `::reveal{character="bianca"}` | `::set{scene.cast.bianca.sealed = false}` |
| `::wear{character="bianca" costume="casual"}` | `::set{scene.cast.bianca.costume = 'casual'}` |

Attribute schemas: `character` (REQUIRED, `providerRef: character`); `::wear` `costume` (REQUIRED,
`enum` ‚àà that character's `registry.costumes`). Raw `::set{scene.cast.*}` remains the low-level
escape hatch; conditional changes use `<match>`/`<branch>` arms containing these.

## 9. Diagnostics (normative lints)

- **Forgot-to-seal:** a character that undergoes a `::reveal` (a `sealed` true‚Üífalse transition)
  but first appears without being sealed (`sealed: true` in `cast:` or a dominating `::seal`) ‚Üí a
  warning (possible name spoiler).
- **Path-dependent voiceBank:** a coded line whose `voiceBank` is path-dependent/ambiguous (¬ß7.3)
  ‚Üí error.
- **Unknown costume / character:** `costume` ‚àâ `registry.costumes`, or a `character` ‚àâ registry ‚Üí
  error (fresh snapshot) or *catalog-stale* diagnostic (stale snapshot).
- **`<match on="scene.cast.<id>.sealed">`** is a `bool` ‚Äî a finite domain; ordinary exhaustiveness
  applies (no `unset`/null case).

## 10. Migration (non-breaking)

Legacy bard authoring is rewritten by the formatter with deprecation warnings; `code=` values are
**never** changed (voice joins stay stable).

| Legacy | Rewritten | voiceKey |
|---|---|---|
| `:line[???-bianca]{code=‚Ä¶}` | `:line[bianca]{code=‚Ä¶}` + `cast.bianca.sealed: true` (or `::seal`) | unchanged (`bianca-‚Ä¶`) |
| `:line[???]` (no id) | requires an explicit id ‚Äî flagged for author fix | n/a |
| `:line[ann_child]{code=‚Ä¶}` | `:line[ann]{code=‚Ä¶}` + `cast.ann.costume: child` + `registry.ann.costumeVoiceBank.child = ann_child` | preserved (`ann_child-‚Ä¶`) |
| `:line[nera_formal]{code=‚Ä¶}` | `:line[nera]{code=‚Ä¶}` + `cast.nera.costume: formal` + `costumeVoiceBank.formal = nera_formal` | preserved (`nera_formal-‚Ä¶`) |

The `-`/`_` bracket-delimiter conventions and the "post-hyphen id" rule are removed; the bracket is
identity only.

## 11. Conformance

A conforming document/registry under this proposal:

- references only registry-known `character` ids and, for `costume`, only that character's
  `registry.costumes`;
- keeps `:line[<id>]` free of delimiter-encoded label/costume;
- mutates cast state only through `::set`/`::seal`/`::reveal`/`::wear` (enum/bool assignments; no
  `unset`);
- resolves sprite/label/voiceKey per ¬ß7, with **path-stable** voiceKey (¬ß7.3).

A conforming consumer resolves these against the one capability snapshot (registry provider + cast
plugin) and reports the ¬ß9 diagnostics.

## Appendix B ‚Äî Open items
