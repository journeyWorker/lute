# Lute Character & Cast ŌĆö Capability Proposal

- **Version:** 0.0.1
- **Status:** Draft (pre-implementation).
- **Scope:** the normative **language contract** for character identity, display label, costume,
  name-reveal, and voice-join in a `.lute` scenario ŌĆö as a **capability plugin** (`character/cast`).
  Covers the character **registry** (provider), the `cast:` **frontmatter**, the `scene.cast.<id>.*`
  **state**, the compile-time **resolution** of sprite / display-name / voiceKey, the `seal`/
  `reveal`/`wear` **directives**, and **migration** of legacy content.
- **Out of scope:** the compiler/runtime **implementation** (deferred; a future Rust core ŌĆö this
  document specifies *what* it must produce, not *how*). The design **rationale + audit trail** is
  [`design.md`](design.md).
- **Depends on:** Lute Scenario DSL `^0.0.1` ([`../scenario-dsl/0.0.1.md`](../scenario-dsl/0.0.1.md))
  with the ┬¦6.1 *plugin-contributed meta keys* amendment and the `as` line attribute; the Lute
  Plugin System `^0.0.1` ([`../plugin-system/0.0.1.md`](../plugin-system/0.0.1.md)) with the
  `frontmatter` export and `assetkinds` (┬¦6.9); plus a `CH` asset kind (asset/core capability)
  with segments `characterId`/`costume`/`emotion`/`variant`.
- **Reference example:** [`../../examples/date-minigame.lute`](../../examples/date-minigame.lute)
  (cast usage to be reflected there when examples are refreshed).

The key words **MUST**, **MUST NOT**, **REQUIRED**, **SHOULD**, **SHOULD NOT**, **MAY** are to be
interpreted as in RFC 2119.

## 1. Abstract

A `:line[<id>]` bracket carries **only** the stable `characterId` (identity ŌĆö the key for sprite
lookup, voice/line joins, and state). Three further concerns are modelled separately, never packed
into the bracket:

1. **display label** ŌĆö the name the dialog box shows (`???`, a mask, or the real name);
2. **costume** ŌĆö an appearance variant of the same identity (the `CH` asset kind's `costume` segment, plugin-system ┬¦6.9);
3. **name-reveal** ŌĆö the `???`ŌåÆreal-name transition, modelled as first-class state that flips once
   and affects all later lines.

All mutable cast state is `enum`/`bool` (satisfying the language's `number|bool|enum` state types,
`../scenario-dsl/0.0.1.md` ┬¦9.3); label strings are **static** configuration; and everything reduces
to flat command records + CEL, preserving the language invariants (total, non-Turing).

## 2. Relationship & dependencies (normative)

This capability is a plugin. It REQUIRES:

- **Language ┬¦6.1 plugin-contributed meta keys** ŌĆö so `cast:` is a legal frontmatter key owned by
  this plugin.
- **Language `:line` `as` attribute** ŌĆö the one-off display-label override (┬¦6).
- **Plugin-system `frontmatter` export** ŌĆö so this plugin can declare and validate the `cast:` key
  (`../plugin-system/0.0.1.md`).
- **Plugin-system providers, state templates, directives, and named lowering hooks** ŌĆö for the

A conforming consumer (checker/LSP/compiler) MUST validate a document against the single resolved
capability snapshot that includes this plugin.

## 3. Character registry (provider)

The registry is global, static character identity data, supplied as a plugin **provider**
(snapshot-first, `../plugin-system/0.0.1.md` ┬¦10). Per character id:

```yaml
bianca:
  name: { ko: "ļ╣äņĢÖņ╣┤", en: "Bianca" }   # REQUIRED ŌĆö canonical display name per language
  costumes:                               # REQUIRED ŌĆö each costume carries its OWN emotion/variant domain
    - { key: default,  emotions: [neutral, content, shy], variants: [0, 1] }
    - { key: waitress, emotions: [content, delighted, surprised], variants: [0, 1] }
  defaultCostume: default                 # REQUIRED ŌĆö a costume `key`
ann:
  name: { ko: "ņĢż", en: "Ann" }
  costumes: [ { key: default, emotions: [delighted, puzzled], variants: [0, 1] },
              { key: child,   emotions: [delighted, puzzled], variants: [0] } ]
  defaultCostume: default
  costumeVoiceBank: { child: ann_child }  # OPTIONAL ŌĆö per-costume static voice bank (┬¦7.3)
fixer: { kind: player }                   # POV/protagonist ŌĆö no sprite, name = {{userName}} (┬¦3.1)
store_staff: { kind: npc, name: { ko: "ņĀÉņøÉ", en: "Staff" } }   # spriteless NPC (┬¦3.1)
emotionGroups:                            # shared; the CH assetKind `emotionGroup` fallback reads this
  joy: { rep: delighted, members: [delighted, content] }
```

| Field | Req. | Rule |
|---|---|---|
| `kind` | MAY | `character` (default) \| `player` \| `npc` ŌĆö see ┬¦3.1 |
| `name` | MUST* | map language code ŌåÆ display string (*omitted for `player`, which renders `{{userName}}`) |
| `costumes` | MUST* | list of `{ key, emotions[], variants[] }`; MUST include a `default` key (*omitted for `player` / spriteless `npc`) |
| `defaultCostume` | MUST* | a costume `key` (*n/a for spriteless) |
| `costumeVoiceBank` | MAY | map costume key ŌåÆ static voice-bank id (┬¦7.3) |
| `emotionGroups` | MAY | top-level (not per-character): `{ group: { rep, members[] } }` for the `emotionGroup` fallback |

The `character` id domain is owned by this provider. A `:line`/`::auto`/`cast:` reference to an id
absent from a **fresh** registry snapshot is a static error; absent from a **stale** snapshot is a
*catalog-stale* diagnostic, not a hard error (`../plugin-system/0.0.1.md` ┬¦10).

### 3.1 Speaker kinds (player ┬Ę character ┬Ę npc)

`kind` distinguishes three speaker kinds:

- **`character`** (default) ŌĆö a full sprite-bearing character (name + costumes); ┬¦5ŌĆō┬¦9 apply.
- **`player`** ŌĆö the POV/protagonist (Bard's `fixer`/`{{user}}`, named by `pov` frontmatter). It has
  **no sprite** and **no `cast` state**; its display label renders as the runtime **`{{userName}}`**
  (not a registry name), and its `voiceKey`/lineId normalize to one canonical id (Bard maps
  `{{user}}` ŌåÆ `fixer` so player lines share a counter). `seal`/`reveal`/`wear` do not apply.
- **`npc`** ŌĆö a **spriteless** speaker (no `costumes`): `:line[store_staff]` emits a speaker+text line
  with **no sprite** and no stage bookkeeping (an off-stage / bit-part voice). It MAY carry a `name`
  (else the id is the label). Ad-hoc unregistered ids are a lint (register even lightly).

A `character` with an empty costume set is likewise **spriteless** (voice-only). Only sprite-bearing
kinds resolve a `CH` assetId (┬¦7.1); player / npc / spriteless skip sprite resolution.

## 4. `cast:` frontmatter (plugin-owned)

`cast:` is a frontmatter meta key owned by this plugin (legal via language ┬¦6.1 + plugin-system
`frontmatter` export). It seeds per-episode initial per-character state and static labels:

```yaml
cast:
  bianca:
    costume: waitress                 # OPTIONAL enum Ōłł registry.costumes; default registry.defaultCostume
    sealed: true                      # OPTIONAL bool; default false
    maskAs: "ņØĆļ░£ņØś ņóģņŚģņøÉ"           # OPTIONAL static string; default "???"
  takeru: { costume: default }         # sealed omitted ŌåÆ false (real name); maskAs unused
```

- Each key MUST be a `character` id present in the registry (┬¦3); an unknown id is a static error.
- `costume` (if present) MUST be a member of that character's `registry.costumes`.
- `sealed` and `costume` seed the corresponding **state** (┬¦5). `maskAs` is **static** (not state).
- A character not listed in `cast:` uses defaults (`costume = defaultCostume`, `sealed = false`,
  `maskAs = "???"`).

## 5. Cast state (normative)

This plugin declares, as a state template keyed by the registry's `character` id domain
(`../plugin-system/0.0.1.md` ┬¦6.3/┬¦7.4), for every referenced character `<id>`:

- `scene.cast.<id>.costume : enum[<registry.costumes for id>]` ŌĆö default `registry.defaultCostume`,
  overridden by `cast:` (┬¦4).
- `scene.cast.<id>.sealed : bool` ŌĆö default `false`, overridden by `cast:` (┬¦4).

Both are `enum`/`bool`, satisfying the language state types (┬¦9.3). They are **`scene`-scoped**
(episode lifetime; reset at episode end, re-materializing defaults). Because `sealed` defaults
`false`, a later episode that omits a character from `cast:` shows the real name automatically;
cross-episode "already known" needs no persisted flag. A rare multi-episode masked arc MAY bind the
mask to a declared `run.*` fact instead (author-declared; outside the auto-declared template).

`maskAs` is **static** per-character-per-episode data from `cast:` (or the default `"???"`); it is
**not** a state path and MUST NOT be written by `::set`.

Cast state is mutated only by `::set` (`../scenario-dsl/0.0.1.md` ┬¦7.3.4) or its sugar (┬¦8). No
`unset`/`null`/delete operation is introduced.

## 6. `:line` display-label override (`as`)

The language `:line` gains an OPTIONAL `as` attribute (a `String`): a one-off display-label override
for that single line. It does not change state. It has the highest precedence in name resolution
(┬¦7.2). Absent `as`, the resolution falls to state + registry.


## 7. Resolution (normative)

For each `:line[<id>]{code, emotion?, variant?, as?}` (and, for sprite/costume, each `::auto`), a
conforming compiler MUST resolve three outputs.

### 7.1 Sprite assetId
Resolve via the `CH` **asset kind** (plugin-system ┬¦6.9): compose its segments with
`characterId = <id>`, `costume =` the character's **current** `scene.cast.<id>.costume` (carried in
stage state), `emotion =` the prior sprite emotion / `neutral`, and `variant = 0` by default. The id
string (e.g. `CH.bianca.waitress.content.0`) is that asset kind's serialization; missing assets
resolve through the kind's declared `fallback`. The current costume MUST apply to **dialogue**
sprites as well as `::auto`, not only where costume is set.

**Only sprite-bearing kinds** (`character` with costumes) resolve a sprite; `player` / `npc` /
spriteless skip ┬¦7.1. `emotion` and `variant` validate against the **current character+costume's**
declared `emotions`/`variants` (per-costume domains, ┬¦3), not a flat enum; an out-of-domain
`emotion`/`variant` resolves through the `CH` assetKind's declared `fallback` (plugin-system ┬¦6.9),
not a raw `neutral`.

### 7.2 Display label
Precedence, first match wins:

1. `kind: player` ŌåÆ the runtime **`{{userName}}`** (never a registry name or mask);
2. the line's `as` attribute (┬¦6), if present;
3. else if `scene.cast.<id>.sealed == true` ŌåÆ `maskAs` (static label; default `"???"`);
4. else ŌåÆ `registry.name[lang]` ŌĆö or, for a `kind: npc` without a `name`, the id itself.

### 7.3 voiceKey (path-stable)
`voiceKey = {voiceBank}-{code}` where `voiceBank` is a **static, compile-time constant**:

- `registry.costumeVoiceBank[costume]` **only when** the character's costume is statically
  determinable at that line (from `cast:` or a dominating static assignment);
- otherwise `characterId`.

`voiceKey` MUST NOT be derived from mutable `scene.cast.<id>.costume` in a way that varies by
branch/timeline path ŌĆö voice joins MUST be path-stable. (Sprites MAY vary per path; they are not
join keys.) If a coded line's `voiceBank` would be path-dependent or ambiguous, it is a static error
(┬¦9); the author pins it or accepts `characterId`. With no `costumeVoiceBank`, `voiceKey =
{characterId}-{code}`.

For a `kind: player` speaker, `voiceKey` normalizes to one canonical id (Bard: `{{user}}` ŌåÆ `fixer`)
so all player lines share a counter, regardless of the authored alias.

## 8. Directives (plugin-provided sugar)

Thin, pure desugaring to `::set` (┬¦7.3.4) ŌĆö no new semantics; the primary authoring surface:

| Directive | Desugars to |
|---|---|
| `::seal{character="bianca"}` | `::set{scene.cast.bianca.sealed = true}` |
| `::reveal{character="bianca"}` | `::set{scene.cast.bianca.sealed = false}` |
| `::wear{character="bianca" costume="casual"}` | `::set{scene.cast.bianca.costume = 'casual'}` |

Attribute schemas: `character` (REQUIRED, `providerRef: character`); `::wear` `costume` (REQUIRED,
`enum` Ōłł that character's `registry.costumes`). Raw `::set{scene.cast.*}` remains the low-level
escape hatch; conditional changes use `<match>`/`<branch>` arms containing these.

## 9. Diagnostics (normative lints)

- **Forgot-to-seal:** a character that undergoes a `::reveal` (a `sealed` trueŌåÆfalse transition)
  but first appears without being sealed (`sealed: true` in `cast:` or a dominating `::seal`) ŌåÆ a
  warning (possible name spoiler).
- **Path-dependent voiceBank:** a coded line whose `voiceBank` is path-dependent/ambiguous (┬¦7.3)
  ŌåÆ error.
- **Unknown costume / character:** `costume` Ōłē `registry.costumes`, or a `character` Ōłē registry ŌåÆ
  error (fresh snapshot) or *catalog-stale* diagnostic (stale snapshot).
- **`<match on="scene.cast.<id>.sealed">`** is a `bool` ŌĆö a finite domain; ordinary exhaustiveness
  applies (no `unset`/null case).

## 10. Migration (non-breaking)

Legacy bard authoring is rewritten by the formatter with deprecation warnings; `code=` values are
**never** changed (voice joins stay stable).

| Legacy | Rewritten | voiceKey |
|---|---|---|
| `:line[???-bianca]{code=ŌĆ”}` | `:line[bianca]{code=ŌĆ”}` + `cast.bianca.sealed: true` (or `::seal`) | unchanged (`bianca-ŌĆ”`) |
| `:line[???]` (no id) | requires an explicit id ŌĆö flagged for author fix | n/a |
| `:line[ann_child]{code=ŌĆ”}` | `:line[ann]{code=ŌĆ”}` + `cast.ann.costume: child` + `registry.ann.costumeVoiceBank.child = ann_child` | preserved (`ann_child-ŌĆ”`) |
| `:line[nera_formal]{code=ŌĆ”}` | `:line[nera]{code=ŌĆ”}` + `cast.nera.costume: formal` + `costumeVoiceBank.formal = nera_formal` | preserved (`nera_formal-ŌĆ”`) |

The `-`/`_` bracket-delimiter conventions and the "post-hyphen id" rule are removed; the bracket is
identity only.

## 11. Conformance

A conforming document/registry under this proposal:

- references only registry-known `character` ids and, for `costume`, only that character's
  `registry.costumes`;
- keeps `:line[<id>]` free of delimiter-encoded label/costume;
- mutates cast state only through `::set`/`::seal`/`::reveal`/`::wear` (enum/bool assignments; no
  `unset`);
- resolves sprite/label/voiceKey per ┬¦7, with **path-stable** voiceKey (┬¦7.3).

A conforming consumer resolves these against the one capability snapshot (registry provider + cast
plugin) and reports the ┬¦9 diagnostics.
